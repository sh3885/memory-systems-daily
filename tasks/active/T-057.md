# Objective

Make repeated delivery of an identical Markdown upload idempotent even when it
arrives with a different Telegram update ID.

# Result

The Telegram draft path now compares the canonical content hash against the
current revision before creating a new revision. A conflict during simultaneous
creation performs the same comparison after reloading current state.
